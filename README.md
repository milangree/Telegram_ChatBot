# 🤖 Telegram_ChatBot

基于 **Cloudflare Pages + Pages Functions + KV / D1 + Vue 3** 的 Telegram 双向消息机器人与 Web 管理后台。

项目目标是提供一套 **可直接部署、可视化管理、支持人机验证、支持论坛话题转发、支持多语言与 2FA 登录** 的 Telegram Bot 管理系统。  
用户向 Bot 私聊发送消息后，机器人会自动在管理员论坛群组中创建对应话题；管理员在话题中的回复会被实时转发回用户，实现完整的双向沟通链路。

---

## ✨ 项目特性

### 核心消息能力
- 📨 **双向消息转发**
  - 用户私聊 Bot
  - 自动在论坛群组内创建独立话题
  - 管理员在话题中回复
  - 自动转发回用户
- 🧵 **按用户 UID 创建独立话题**
  - 便于区分不同用户
  - 话题命名附带 UID，减少重名混淆
- 📝 **管理员编辑消息同步**
  - 在一定时间窗口内，管理员修改已发送消息时，可同步撤回/重发给用户
- 🧹 **消息去重**
  - 后端处理去重，避免重复转发
  - 前端会话显示去重，避免 WebUI 中重复显示相同消息
- 🗑 **删除会话并关闭话题**
  - 删除历史消息
  - 关闭对应群组话题
  - 非白名单用户会在下次对话时重新验证

### 反骚扰与风控能力
- 🔐 **人机验证**
  - 数学题按钮验证
  - 图片验证码（4 位数字）
  - 图片验证码（5 位字母数字）
- 🚦 **频率限制**
  - 支持设置每分钟最大消息数
- 🚫 **封禁管理**
  - 封禁 / 解封 / 永久封禁
  - 白名单跳过验证
  - 支持用户申诉
- 📴 **机器人命令过滤**
  - 可过滤 `/xxx` 指令，不转发给管理员

### WebUI 管理后台
- 📊 **仪表盘**

  - 总消息数
  - 今日消息数
- 💬 **对话记录**
  - 查看用户会话
  - 搜索用户
  - 删除对话
- 👥 **用户管理**
  - 批量封禁 / 解封 / 加白名单
  - 用户详情弹窗
  - 头像、用户名、语言、验证状态展示
- ⚙️ **系统设置**
  - Bot Token
  - Webhook URL
  - 图片验证码站点 URL
  - 管理员 Telegram ID
  - 白名单、申诉、验证、自动撤回等功能开关
  - 存储切换（KV / D1）
  - 清空机器人业务数据（保留 WebUI 登录账号）

  - 登录页与后台均支持切换

### 安全与认证

## 🧭 系统工作流程

### 用户消息流程
1. 用户向 Bot 私聊发送消息
2. Bot 检查：
   - 是否已封禁
   - 是否需要人机验证
   - 是否超过频率限制
3. Bot 将消息转发到管理员论坛群组
4. 若该用户没有对应话题，则自动创建新话题
5. 管理员在话题内回复
6. 回复被转发回用户
7. 消息记录写入存储（KV / D1）

### 管理员消息流程
1. 管理员在论坛话题中发送消息
2. 系统根据 thread 映射定位用户
3. 将消息发送给对应 Telegram 用户
4. 记录管理员消息与映射关系
5. 若管理员后续编辑消息，则在允许时间窗口内同步更新给用户

## 🚀 快速部署教程

## 1. Fork 或导入仓库

将本仓库 Fork 到你自己的 GitHub 账号，或直接克隆后推送到你自己的仓库。

---

## 2. 准备 Cloudflare 资源

### 必需资源：KV
登录 Cloudflare Dashboard 后创建一个 KV 命名空间：

1. 进入 **Workers & Pages**
2. 打开 **KV**
3. 点击 **Create a namespace**
4. 名称可自定义，例如：

```text
tg-chatbot-kv
```

### 可选资源：D1
项目默认可直接使用 KV。  
如果你希望使用 SQL 数据库，可以额外创建 D1，并在 Pages 项目中绑定后，在系统设置中切换到 D1。

---

## 3. 创建 Cloudflare Pages 项目

1. 打开 **Workers & Pages**
2. 点击 **Create**
3. 选择 **Pages**
4. 连接 GitHub 仓库
5. 选择本项目仓库
6. 填写构建参数：

| 配置项 | 值 |
|---|---|
| 框架预设 | `Vue` |
| 构建命令 | `npm run build` |
| 构建输出目录 | `dist` |

7. 开始部署

> 如果第一次部署失败，通常是因为还没有绑定 KV，这属于正常现象，继续下一步即可。

---

## 4. 添加绑定（Bindings）

### 绑定 KV
在 Pages 项目中进入：

**Settings → Bindings → Add binding**

类型选择 **KV Namespace**，并填写：

| 字段 | 值 |
|---|---|
| Variable name | `KV` |
| Namespace | 你创建的 KV 命名空间 |

> `KV` 变量名必须保持一致，不能改成其他名称。

### 可选：绑定 D1
如果你创建了 D1，也可以继续添加：

| 字段 | 值 |
|---|---|
| Variable name | `D1` |
| Database | 你的 D1 数据库 |

---

## 5. 重新部署

完成 KV（和可选 D1）绑定后，重新部署 Pages 项目。

成功后你会得到一个站点地址，例如：

```text
https://your-project.pages.dev
```

## 🗃 数据存储说明

项目支持两种后端存储：

### 1. KV
优点：
- 配置简单
- 部署方便
- 对小中型机器人足够实用

适合：
- 轻量部署
- 快速上线
- 免费额度优先

### 2. D1
优点：
- SQL 查询能力更强
- 更适合后期扩展与数据分析

适合：
- 对数据结构化要求更高
- 有更多统计、筛选、管理需求

### 存储切换
系统支持 **KV 与 D1 之间同步后切换**。  
切换时会自动迁移数据，减少手动操作成本。

## ❓常见问题（FAQ）

## 1. 打开 WebUI 返回 500 或白屏
请优先检查：
- 是否已绑定 `KV`
- 变量名是否为大写 `KV`
- 是否完成重新部署

---

## 2. 设置 Webhook 失败
请检查：
- Bot Token 是否正确
- Webhook URL 是否为公网 HTTPS 地址
- 地址格式是否为：

```text
https://你的域名/webhook
```

---

## 3. 用户消息没有出现在论坛群组
请检查：
- 论坛群组是否开启“话题”
- Bot 是否已加入该群组
- Bot 是否拥有创建/管理话题权限
- `FORUM_GROUP_ID` 是否正确（超级群组一般以 `-100` 开头）

---

## 4. 找不到用户头像或管理员资料
这通常是因为：
- 该用户尚未与 Bot 交互
- 无法从本地用户表中搜索到对应记录
- Telegram 侧用户资料不可直接获取完整信息

系统会自动回退显示为：
- 首字母头像
- 用户名或 ID

---

## 5. 忘记了 WebUI 密码怎么办？
- 如果账号启用了 2FA，可通过登录页“找回密码”重置
- 如果没有启用 2FA，则需要手动处理存储中的 Web 用户数据

---

## 6. 清空数据库会删除登录账号吗？
不会。  
系统的“清空数据库”功能只会清空机器人产生的业务数据，例如：
- 用户
- 消息
- 白名单
- 验证状态
- 会话数据等

**WebUI 登录账号和 2FA 信息会保留。**

---

## 7. 免费套餐够用吗？
对于轻量到中等规模的 Bot，一般足够。  
具体仍需根据你的消息量、图片验证码使用频率和 API 调用频率评估。

## 🛡 安全说明

- 登录会话使用 Token 机制管理
- 支持 2FA（TOTP）

- 支持管理员与用户消息映射，降低串话风险
- 支持消息去重，减少重复发送问题
- 支持清除旧 Webhook，减少切换 Bot Token 后的重复更新风险
